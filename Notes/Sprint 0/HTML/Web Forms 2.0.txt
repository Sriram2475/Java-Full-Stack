🌐 Web Forms 2.0
Web Forms 2.0 is an improvement over the traditional HTML forms, mainly introduced with HTML5.
Its goal is to enhance form controls, improve data validation, reduce the need for JavaScript, and improve user experience.
Web Forms 2.0 added:

New input types
Built‑in validation
Placeholder text
Autofocus
Required fields
Output element
Improved UI controls (date pickers, sliders, color pickers)

📝 <input> Element in HTML5

HTML5 expanded the <input> element by introducing new input types to make form creation easier and validation more automatic.

✔ Common HTML5 Input Types

Type - Purpose
email - Validates email format
url - Validates URL
number - Numeric input with min/max/step
date, month, week - Calendar selectors
time, datetime-local - Time input
range - Slider control
color - Color picker
tel - Telephone number (no automatic validation)
search - Search box with styling

Example:
<input type="email" name="userEmail" required>
<input type="number" min="1" max="10">
<input type="date">
<input type="color">

🟣 <output> Element
The <output> element represents calculated results in a form.
It is often used with JavaScript.
Example:
<form oninput="result.value = parseInt(a.value) + parseInt(b.value)">
  <input type="number" id="a" value="5"> +
  <input type="number" id="b" value="10"> =
  <output name="result"></output>
</form>

🟡 placeholder Attribute
The placeholder attribute shows light‑gray hint text inside an input box until the user types something.
Example:
<input type="text" placeholder="Enter your full name">
Important: Placeholder is not a label and should not replace one.

🔵 autofocus Attribute
The autofocus attribute automatically places the cursor in that field when the page loads.
Example:
<input type="text" autofocus>
Notes:
Only one element in a page should have autofocus

🔴 required Attribute
The required attribute ensures the user must fill the field before submitting the form.
Example:
<input type="email" required>
