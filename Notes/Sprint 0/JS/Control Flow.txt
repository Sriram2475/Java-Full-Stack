Control flow

1. Conditional Statements
Conditional statements allow the program to make decisions.

A. if Statement
Executes code only if a condition is true.

let age = 20;

if (age >= 18) {
    console.log("You are an adult");
}

B. if-else Statement
Adds an alternative.
let marks = 45;

if (marks >= 50) {
    console.log("Pass");
} else {
    console.log("Fail");
}

C. if-else if Ladder
let temp = 30;

if (temp > 40) {
    console.log("Very Hot");
} else if (temp > 30) {
    console.log("Hot");
} else if (temp > 20) {
    console.log("Warm");
} else {
    console.log("Cold");
}

D. switch Statement
Best when checking multiple fixed values.

let day = 3;

switch (day) {
    case 1:
        console.log("Monday");
        break;
    case 2:
        console.log("Tuesday");
        break;
    case 3:
        console.log("Wednesday");
        break;
    default:
        console.log("Invalid day");
}

2. Loops
Loops repeat a block of code until a condition becomes false.

A. for Loop
Used when you know the number of iterations.
for (let i = 1; i <= 5; i++) {
    console.log("Count:", i);
}

B. while Loop
Runs as long as condition is TRUE.
let i = 1;

while (i <= 5) {
    console.log(i);
    i++;
}

C. do-while Loop
Executes at least once even if condition is false.
let i = 1;

do {
    console.log(i);
    i++;
} while (i <= 5);


D. for...of Loop
Iterate over values of an iterable
(arrays, strings, Maps, Sets, etc.)
let fruits = ["Apple", "Banana", "Mango"];
for (let fruit of fruits) {
console.log(fruit);
}
Use for...of when you want each value from an array or other iterable.

E. for...in loop — iterate over enumerable keys (object properties)
Iterates over object properties.

let person = { name: "Sriram", age: 22 };

for (let key in person) {
    console.log(key, person[key]);
}
Use for...in for plain objects. Avoid using it on arrays (it iterates keys as strings and can include inherited properties).

What about forEach? Do we use it?
Yes—on arrays only. It runs a callback for each item.
const fruits = ["Apple", "Banana", "Mango"];

fruits.forEach((fruit, index, arr) => {
  console.log(index, fruit);
});
// Output:
// 0 Apple
// 1 Banana
// 2 Mango

⚠️ Important differences & pitfalls


break / continue

✅ Works in for, while, do...while, and for...of
❌ Does NOT work in forEach

// for...of supports break
for (const fruit of ["Apple", "Banana", "Mango"]) {
  if (fruit === "Banana") break; // works
  console.log(fruit);
}

// forEach can't break/continue
["Apple", "Banana", "Mango"].forEach(fruit => {
  if (fruit === "Banana") return; // only returns from callback, not from the loop
  console.log(fruit);
});


Async/Await

✅ for...of runs sequentially with await
⚠️ forEach does not await the callback—promises run concurrently and can be confusing

// Correct sequential async iteration
for (const url of urls) {
  const res = await fetch(url);
  console.log(await res.text());
}

// Anti-pattern: forEach + async
urls.forEach(async (url) => {
  const res = await fetch(url); // not awaited at loop level
});


Objects vs Arrays

✅ Use for...in for objects
✅ Use for...of or forEach for arrays
If you want both key and value from an object:

const person = { name: "Sriram", age: 22 };

// Option 1: keys
Object.keys(person).forEach(key => {
  console.log(key, person[key]);
});

// Option 2: entries (key/value pairs)
for (const [key, value] of Object.entries(person)) {
  console.log(key, value);
}

Maps/Sets

const map = new Map([
  ["name", "Sriram"],
  ["role", "Intern"]
]);

for (const [key, value] of map) {
  console.log(key, value);
}
// forEach works too on Map/Set
map.forEach((value, key) => console.log(key, value));

Quick rule of thumb

Array values → for...of (or forEach if you don’t need break/continue/await)
Array indices → traditional for (let i = 0; i < arr.length; i++)
Object properties → for...in (with hasOwnProperty) or Object.keys/entries + for...of/forEach
Async sequential → for...of
Early exit needed → for...of (or classic for)

3. Error Handling
JavaScript errors are handled using try-catch.

A. Basic Error Handling
try {
    let a = undefinedVariable;  // error
} catch (error) {
    console.log("Error occurred:", error.message);
}

B. finally Block
Runs whether an error occurs or not.
try {
    console.log("Trying...");
} catch (error) {
    console.log("Error:", error);
} finally {
    console.log("Always executes");
}

C. Throwing Custom Errors
You can create your own errors.
function divide(a, b) {
    if (b === 0) {
        throw new Error("Cannot divide by zero!");
    }
    return a / b;
}

try {
    divide(10, 0);
} catch (e) {
    console.log(e.message);
}
