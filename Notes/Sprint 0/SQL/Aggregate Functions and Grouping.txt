Aggregate Functions and Grouping

⭐ 1. Introduction to Aggregate Functions
Aggregate functions perform calculations on multiple rows and return a single value.
They are used for:

Summaries
Reports
Analytics
Statistics

✔ Common Aggregate Functions
1. SUM() – Adds values
Adds all values of a numeric column.
SELECT SUM(salary) AS total_salary
FROM employees;

2. AVG() – Calculates average
Returns the average value from a numeric column.
SELECT AVG(salary) AS avg_salary
FROM employees;

3. COUNT() – Counts rows
Counts how many rows or non-null values exist.

Count all rows:
SELECT COUNT(*) FROM employees;

Count non-null values:
SELECT COUNT(salary) FROM employees;

4. MIN() – Returns smallest value
SELECT MIN(salary) FROM employees;

5. MAX() – Returns largest value
SELECT MAX(salary) FROM employees;

⭐ 2. GROUP BY Clause
GROUP BY is used to group rows that have the same values in one or more columns.
Why use GROUP BY?
To get aggregate results for each group.
Syntax:
SELECT column1, aggregate_function(column2)
FROM table_name
GROUP BY column1;

✔ Example: Total salary by department

SELECT department, SUM(salary) AS total_salary
FROM employees
GROUP BY department;

⭐ 3. HAVING Clause
HAVING is like WHERE, but used after grouping.
Difference:

WHERE - HAVING
Filters rows before GROUP BY - Filters groups after GROUP BY
Cannot use aggregate functions - Can use aggregate functions

✔ Example without HAVING:
SELECT department, SUM(salary)
FROM employees
GROUP BY department;
``

✔ Example with HAVING:
Show only departments whose total salary > 100000
SELECT department, SUM(salary) AS total_salary
FROM employees
GROUP BY department
HAVING SUM(salary) > 100000;

⭐ 4. Combining WHERE + GROUP BY + HAVING
Important:

WHERE filters rows
GROUP BY groups remaining rows
HAVING filters groups

✔ Practical Example
Get total salary of IT and HR departments, but only show groups where total salary > 80,000


SELECT department, SUM(salary) AS total_salary
FROM employees
WHERE department IN ('IT', 'HR')          -- filter rows
GROUP BY department                       -- group
HAVING SUM(salary) > 80000;               -- filter groups


Problem:
In an organization, find:

Number of employees in each department
Average salary for each department
Show only departments with more than 4 employees
Sort by average salary (high → low)

Query:

SELECT
    department,
    COUNT(*) AS employee_count,
    AVG(salary) AS avg_salary
FROM employees
GROUP BY department
HAVING COUNT(*) > 4
ORDER BY avg_salary DESC;


⭐ 1. Pattern Matching With LIKE Operator
The LIKE operator is used for searching text patterns.
✔ % → matches any number of characters
Examples:
WHERE name LIKE 'A%'      -- starts with A
WHERE name LIKE '%n'      -- ends with n
WHERE name LIKE '%an%'    -- contains 'an'
✔ _ → matches exactly one character
Examples:
WHERE name LIKE '_at'     -- cat, bat, hat (3-letter ending with 'at')
WHERE emp_id LIKE '12_5'  -- matches 1235, 1245 etc.


⭐ 2. Common String Functions in SQL
These functions are essential for manipulating and filtering text.

✔ 1. SUBSTRING()
Extracts a part of a string.
SELECT SUBSTRING(name, 1, 3) FROM employees;
Gets first 3 characters of name.

✔ 2. LENGTH()
Returns the number of characters.
SELECT LENGTH(name) FROM employees;

✔ 3. UPPER() / LOWER()
Convert text to uppercase or lowercase.
SELECT UPPER(name) FROM employees;
SELECT LOWER(name) FROM employees;

✔ 4. CONCAT()
Joins two strings together.
SELECT CONCAT(first_name, ' ', last_name) AS full_name
FROM employees;

✔ 5. TRIM() / LTRIM() / RTRIM()
Remove spaces.
SELECT TRIM(name) FROM employees;

✔ 6. REPLACE()
Replace text inside a string.
SELECT REPLACE(name, 'a', '@') FROM employees;

✔ 7. LEFT() / RIGHT()
Extract characters from left or right.
SELECT LEFT(name, 2) FROM employees;   -- first 2 letters
SELECT RIGHT(name, 2) FROM employees;  -- last 2 letters

⭐ 3. Common Numeric Functions
Useful for calculations and analytics.

✔ 1. ROUND()
Rounds a number.
SELECT ROUND(salary, 0) FROM employees;

✔ 2. CEIL() / FLOOR()
SELECT CEIL(45.2);   -- 46
SELECT FLOOR(45.8);  -- 45

✔ 3. ABS()
Absolute value.
SELECT ABS(-10);   -- 10

⭐ 4. Date Functions (Very Important for Projects)

✔ 1. NOW()
Returns current date & time.
SELECT NOW();

✔ 2. CURDATE()
Returns today’s date.
SELECT CURDATE();

✔ 3. DATE_ADD() / DATE_SUB()
Add or subtract days/months/years.
SELECT DATE_ADD(CURDATE(), INTERVAL 7 DAY);

✔ 4. YEAR(), MONTH(), DAY()
Extracts parts of a date.
SELECT YEAR(hire_date), MONTH(hire_date)
FROM employees;

⭐ 1. LIMIT
✔ Purpose
To restrict the number of rows returned by a query.
✔ Syntax
SELECT columns
FROM table
LIMIT n;
✔ Example – Get first 5 rows
SELECT * FROM employees
LIMIT 5;

Output
Returns only 5 rows, even if the table has 500 rows.

⭐ 2. OFFSET
✔ Purpose
To skip a certain number of rows before starting to return results.
✔ Syntax
SELECT *
FROM table
LIMIT n OFFSET m;

m → number of rows to skip
n → number of rows to fetch after skipping

✔ Example – Skip first 5 rows, return next 5 rows
SELECT * FROM employees
LIMIT 5 OFFSET 5;
This fetches rows 6 to 10.

⭐ 3. LIMIT with OFFSET — Used for Pagination
Example:
If each page shows 10 records
✔ Page 1
LIMIT 10 OFFSET 0;
✔ Page 2
LIMIT 10 OFFSET 10;

⭐ 4. MySQL Shortcut Format
Instead of LIMIT n OFFSET m, MySQL allows:
LIMIT offset, count;

✔ Example
SELECT * FROM employees
LIMIT 5, 10;
This means:

Skip first 5 rows
Return next 10 rows

⭐ 5. ORDER BY is Important When Using LIMIT
Because the database has no guaranteed order, ALWAYS combine with ORDER BY.
Bad (random results):
SELECT * FROM employees LIMIT 5;
Good:
SELECT * FROM employees
ORDER BY salary DESC
LIMIT 5;

⭐ 6. Other Similar Keywords You Should Know
✔ FETCH FIRST (ANSI SQL / PostgreSQL / Oracle)
Used instead of LIMIT.
SELECT *
FROM employees
FETCH FIRST 5 ROWS ONLY;

✔ TOP (SQL Server)
SQL Server does not use LIMIT.
It uses TOP.
SELECT TOP 5 *
FROM employees;

✔ ROW_NUMBER() OVER()
Used for advanced pagination and ranking.
SELECT name, salary,
ROW_NUMBER() OVER (ORDER BY salary DESC) AS rank
FROM employees;
